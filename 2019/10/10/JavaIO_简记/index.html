java类的构造器

有构造器：直接使用new xx()可以创建实例

无构造器：不需要实例，可能是工具类，如(math)类 / 通过静态方法得到实例，如(Runtime)类

分隔符

  1. static String pathSeparator 路径分隔符 Windows(;) Linux(:)
  2. static char pathSeparatorChar 1的字符型
  3. static char Separator 名称分隔符 ()

  4. static char separatorChar 3的字符型

    字符集与乱码

    乱码原因：

  5. 因为初始化String实例时给的字节数组长度不是字节数组长度（如定长unicode四个汉字应该是8个字节，结果给String在初始化时传的长度是7个，那么最后一个字符很可能乱码）

  6. 编码与解码用的字符集不统一

    String msg = new String("姓名生命使命");
byte[] by = msg.getBytes();//编码，这里可以指定编码格式，3有解释

String str = new String(by,0,by.length);
System.out.println(str);
str = new String(by,0,by.length-1);
System.out.println(str);

System.out.println("------------");
str = new String(by,0,by.length,"utf8");//解码
System.out.println(str);
str = new String(by,0,by.length,"gbk");//解码
System.out.println(str);

字节流与字符流

字节流操作的对象是字节byte[]，
字符流操作的对象是字符char[].(另外字符输出流还可以使用write.append()追加输出内容)

节点流与处理流

一般以File，ByteArray前缀的流是节点流
处理流底层是节点流，处理流的目的是简化操作提高效率，缓冲流关闭时会自动关闭它里边的节点流，所以不需要再次关闭缓冲流，缓冲流也分字节缓冲流和字符缓冲流

字节数组流

ByteArrayInputStream,ByteArrayOutputStream(不需要指定目的地，有java开辟内存存储，有新增方法，建议不使用多态来新建实例)，并且他们不需要调用close()释放资源，它们不需要通知操作系统关闭文件，它们的close()函数体是空的
InputStream is = new ByteArrayInputStream(byte[]);
ByteArrayOutputStream os = new ByteArrayOutputStream();//无目的地
